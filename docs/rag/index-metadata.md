# Index Metadata

PureLink records retrieval index metadata in `document_indexes`.

## Vector Index Metadata

For vector indexes, the table records:

- document id
- knowledge base id
- index type
- provider
- model name
- model dimension
- model version
- status
- indexed time

This prevents silent comparison between vectors generated by incompatible embedding models.

## Status

Supported statuses:

- `pending`
- `indexing`
- `indexed`
- `stale`
- `failed`

## Compatibility

Retrieval checks indexed documents against the current embedding provider/model/dim. Legacy documents without metadata remain retrievable for backward compatibility.

## Rebuild Readiness

Changing `EMBEDDING_PROVIDER` or `EMBEDDING_MODEL` requires rebuilding affected indexes. M4 prepares the metadata layer for that workflow; it does not claim a full rebuild UI.
